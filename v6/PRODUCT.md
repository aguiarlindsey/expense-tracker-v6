# Product

## Register

product

## Users

Single user (the developer) using this daily on desktop and mobile. Context: quick expense entry mid-day, deeper review sessions weekly. The user is technically fluent — they built the tool — so the interface can trust them with complexity and skip hand-holding.

## Product Purpose

A full-stack personal finance tracker: log expenses and income, track budgets and savings goals, monitor spending across currencies, and surface anomalies. Success means the user never has to open a spreadsheet or a generic finance app — this does more, faster, and looks better than anything on the market.

## Brand Personality

Polished, modern, premium. The feel of a tool you'd pay for. High craft — every detail earns its place. Not corporate, not generic.

## Anti-references

- **Corporate SaaS** (Salesforce, SAP style): dense grey tables, zero personality, enterprise defaults.
- **Generic finance app** (Mint, YNAB clone): blue-on-white safety, stock chart aesthetic, forgettable.

## Design Principles

1. **Earn every decoration.** If a visual element doesn't help the user understand or act, remove it.
2. **Data is the hero.** Typography, spacing, and layout exist to make numbers scannable and comparable — not to showcase the design system.
3. **Consistent affordances.** Same button, same form control, same icon style everywhere. Delight is in moments, not every surface.
4. **Premium through restraint.** High craft reads as fewer things done extremely well, not more things done adequately.
5. **Accessibility is not optional.** This is a daily-use personal tool — contrast and keyboard nav matter even for a single user.

## Accessibility & Inclusion

Target WCAG AA. Colorblind mode already implemented. Reduced motion support needed. Single user, but standards still apply.
