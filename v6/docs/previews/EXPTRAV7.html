Getting Started

Login

  • Go to https://expense-tracker-v6.vercel.app
  • Enter your email and click Send Magic Link
  • Open the email, click the link — logged in automatically
  • No password required; the magic link expires after 1 use

First-time setup (recommended)

  1. Settings → set your Base Currency (default: INR ₹)
  2. Settings → Budgets → set monthly/daily/weekly spending limits
  3. Settings → Security → enrol biometric (fingerprint / Face ID)
  4. Settings → Recurring Reminders → Enable push notifications
  5. If migrating from V5 → Settings → Migrate from V5

Recurring Tab

What is Recurring?

A template for a repeating expense or income (e.g. rent, Netflix, salary). Stored once with a next due date. Not auto-duplicated — you add it manually when it's due.

Due Badges

Green = upcoming · Amber = due ≤3 days · Red = overdue

Creating a Recurring Entry

When adding any expense or income, tick Recurring and select a period: Daily / Weekly / Bi-weekly / Monthly / Quarterly / Yearly. Next due date is auto-calculated.

Receipt Scanner (OCR) NEW

Scan a receipt photo and automatically pre-fill the expense form. Powered by Tesseract.js running entirely in your browser — no image is ever uploaded to a server.

Opening the Scanner

In the Add Expense form, tap the 📷 Scan button in the top-right of the form header.

Single-Photo Scan

  1. Select 1 Photo
  2. Take a photo or upload from gallery
  3. Drag the 8 crop handles (corners + edges) to frame the receipt
  4. Tap Scan Selected Area or Scan Full Image
  5. Progress bar shows Tesseract reading the text (0–100%)
  6. Review the results screen, then tap Apply to Form

Multi-Part Scan — Long Receipts NEW

For receipts too tall to photograph clearly in one shot:

  1. Select Multi-Part
  2. Photograph top section → crop → tap Add Another Part →
  3. Photograph bottom section → crop → tap Add Another Part → for more, or Scan N Parts to finish
  4. Progress bar divides evenly across all parts
  5. All text is combined in order and parsed as one document
Tip
Start each new part slightly above where the previous one ended — a small overlap ensures no lines are missed. No hard limit on the number of parts.

What the Scanner Extracts

FieldHow it's detected
AmountLooks near "Total", "Bill Total", "Grand Total", "Sale Amount" first; then any ₹/Rs/¥/€/£ symbol
Date8 format patterns: DD/MM/YYYY · DD-MM-YY · ISO · "12 May 2026" · "May 12, 2026" etc.
MerchantAll-caps lines in the top 4 lines (skips company registration suffixes like PVT LTD); falls back to highest-scoring line
Category80+ merchant keyword map; then food/fuel content detection from item names
Payment MethodCash · UPI · GPay · PhonePe · Paytm · VISA · MasterCard · RuPay · Debit Card · MOP CARD · Net Banking
SGST / CGST / IGST / VAT / Service Charge / CessLine-by-line regex for each tax type
Fuel Rate per Litre"Rate Per Litre 103 80" → normalised to 103.80, rounded to 2dp
Fuel Quantity"Sale Quantity 19 36" → normalised to 19.36, rounded to 2dp
Fuel Type"Product Name Petrol / Diesel / CNG / LPG"

Results Screen

  • Each field shows ✓ (found with confidence) or ? (uncertain)
  • Tax chips show individual amounts per tax type
  • Fuel chips show rate, litres, fuel type
  • Show raw OCR text ▼ reveals full extracted text for debugging
  • Re-scan to try again · Apply to Form to pre-fill
First Use
The first scan downloads Tesseract's English language model (~10 MB from CDN) and caches it in your browser. Subsequent scans are fast. Works best with sharp, well-lit photos of printed receipts.

Incognito Mode NEW

Blurs all monetary amounts throughout the app so sensitive figures aren't visible to bystanders.

Toggling

Tap the 🙈 button in the top-right header. All amounts blur immediately across all tabs. Tap again to reveal all.

Revealing Individual Amounts

While in incognito mode, hover (desktop) or tap (mobile) any blurred amount to reveal just that value temporarily.

Persistence

Incognito state is saved to localStorage — stays on until you toggle it off, even across page reloads.

Biometric Lock & OTP Fallback NEW

Enrolment

  1. Settings → Security → Set Up Biometric
  2. Enter your backup email (for OTP recovery — different from login email)
  3. Follow the browser prompt to register fingerprint or Face ID

Lock Screen

Every time you open the app, the lock screen appears. Tap Verify with Biometrics → authenticate → app unlocks instantly.

OTP Fallback

If biometrics fail: tap Use Email Code → 6-digit OTP sent to backup email → enter within 15 minutes. After 3 incorrect attempts, account locks for 15 minutes.

Per-Device Independence

Each device enrols separately. One device's credential does not affect others. Removing biometrics on one device only removes it from that device.

Anomaly Detection NEW

How It Works

When a new expense is saved, it is compared to your past expenses in the same category. If significantly above your historical average, it is flagged.

Anomaly Panel

Appears at the top of the Insights tab. Shows up to 5 most recent anomalies: date, description, category, % above average. Dismiss individually with ✕.

Persistence

Dismissed anomaly IDs stored in localStorage (max 500) — dismissed anomalies don't reappear.

Conflict Resolution NEW

When It Triggers

When the same expense is edited on two devices simultaneously. The second save detects the database version has changed.

Conflict Modal

Side-by-side diff of your local version vs the server version. Three modes:

  • Keep Mine — your local changes win
  • Keep Theirs — the server version wins
  • Merge — per-field radio buttons; choose which version of each field to keep

Conflict Badge

Header shows the count of pending conflicts. Tap to open the conflict resolution modal.

21

Offline Mode

The app works fully offline. No data is lost when connectivity drops.

  • Reading — last-loaded data available; charts and lists work normally
  • Writing — changes go to a local retry queue, appear in the UI immediately
  • Reconnection — queued changes sync to Supabase automatically in dependency order
  • ⏳ badge — on items pending cloud confirmation
  • Amber banner shows "You're offline · N changes queued"
  • Blue banner shows while the queue drains after reconnection
  • Items failing 5 retries are dropped to prevent infinite loops

Push Notifications

Daily reminders for recurring expenses due within 3 days.

Enabling

  1. Settings → Recurring Reminders → Enable
  2. Approve the browser permission prompt

How It Works

Fires at 07:00 AM IST daily. Shows description, amount, days until due. Tapping opens the app to the Recurring tab. Works when the browser/app is closed.

Data Migration from V5

Steps

  1. Open the old V5 HTML file in a browser
  2. Settings → Export Data → Download JSON
  3. In V7: Settings → Migrate from V5 → Choose V5 File
  4. Migration runs automatically — no duplicates created

What Migrates

All expenses and income records (categories, tags, notes, recurring flags, amounts, dates). Fingerprints preserved — re-running is safe.

What Does NOT Migrate

Budgets · Goals · App settings — re-set these manually in V7.
